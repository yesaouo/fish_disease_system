================================================================================
[5] Phase 4 — Case encoder（單向量檢索，論文加速貢獻）
================================================================================
動機：Phase 1 hungarian 每 query 對 case_db 做 multi-vector + Hungarian
匹配（~15 ms/q）。蒸餾成單一 cosine 後快 14×，且加上 case→cause text 的
直接監督後微幅超越老師。

最終選擇配方：DeepSets encoder + dual-target loss（listwise KL distillation
+ case-to-cause SupCon InfoNCE）。**直接用 SDM env 跑，無 mamba_ssm 依賴。**

Mamba 變體現在是 architecture ablation，code 已搬到
`diagnosis_model/cause_inference/mamba_ablation/`（見最後 [5-mamba] 區塊）；
build_encoder lazy-import，只在 `--encoder_type mamba` 時才需要 mamba3 env +
CC=/usr/bin/gcc-12。

/home/lab603/anaconda3/envs/SDM/bin/python \

--------------------------------------------------------------------------------
[5b] 訓練 DeepSets dual-target encoder（最終選擇，~10 分鐘）
--------------------------------------------------------------------------------
Loss = listwise_KL(student case-pair sim, teacher case-pair sim) at T=0.1
     + 0.5 * SupCon(h_final, GT cause text embs) at T=0.07

/home/lab603/anaconda3/envs/SDM/bin/python \
  -m diagnosis_model.cause_inference.train_case_encoder \
  --case_db_dir diagnosis_model/cause_inference/outputs/case_db \
  --teacher_path diagnosis_model/cause_inference/outputs/case_db/teacher_train_train.pt \
  --output_dir diagnosis_model/cause_inference/outputs/encoder_final \
  --encoder_type deepsets \
  --batch_size 256 --epochs 50 \
  --temp_target 0.1 --temp_pred 0.1 \
  --use_infonce --infonce_weight 0.5 --infonce_temp 0.07

訓練輸出：
  best_encoder.pt — best by valid naive R@10
  last_encoder.pt
  train_log.jsonl
  config.json     — 含 encoder_type / dim / hyperparams

備註：in-training 的 sem_R@10 用 case-to-case GT-intersection naive metric（為了
省每個 epoch 跑全 Phase 1-aligned pipeline 的時間）。真實 Phase 1-aligned R@10
要事後用 eval_phase1_aligned.py 評（見 inference.txt [I]）。
