================================================================================
[6] Phase 4 — ABQ (Aggregation-Buffered Quantization)：case-bank codebook 壓縮（RVQ；論文 method-level 貢獻＝aggregation-buffer scaling law）
================================================================================
做法：對 Phase 3 凍結的 DeepSets z 做 codebook 量化（RVQ 為實作）壓縮，選配
residual reranker 學 Δ ≈ qᵀe 修正排序。命名收尾（2026-05-21）：不再用 CRR-DeepRVQ /
DeepRVQ 招牌——RVQ 與 reranker 都是元件。端到端（eval_ceah_compressed.py）顯示
reranker 在 production（top_k=20 + CEAH）多餘，只在 buffer-free（top_k=1）部署有效，
故 reranker 是對照點、非 production 元件；method-level 貢獻是 aggregation-buffer 吸收
量化損害的 scaling law（D ≈ c·ε/K^0.84）+ importance-weighted codebook。

把 Phase 3 已訓好的 DeepSets case encoder 凍結後，對 z_i ∈ R^768 做 post-hoc
Residual Vector Quantization 壓縮成 codes ∈ [K]^M（每 case M bytes for K≤256），
再用 query-to-candidate cross-attention reranker 學 Δ_i ≈ qᵀe_i 修正 RVQ 引入
的 ranking 誤差。

s_first(q, i) = qᵀẑ_i              # LUT scoring on RVQ codes, O(M) per candidate
s_final(q, i) = s_first + Δ(q, i)   # neural reranker on top-K_top candidates

兩個 reranker variant：
  Light  : ẑ, codes, q⊙ẑ, |q-ẑ|, s_first, ‖e‖    （compressed-memory deployment）
  Full   : 同上 + z_i + e_i                       （full-memory rerank;
                                                   analytic Δ = qᵀe 為 oracle）

論文評估在 Phase 1-aligned eval 上跑兩個 regime：
  Regime A : top_k_cases=20  → aggregation buffer 吸收壓縮損失，所有方法持平
  Regime B : top_k_cases=1   → 無 aggregation buffer，這是 method validation regime

子套件路徑：diagnosis_model/cause_inference/rvq_rerank/
產出目錄：outputs/rvq_rerank/

[6c] Stage A3 — 訓練 Light reranker（每個 RVQ config ~15s–2min）
--------------------------------------------------------------------------------
Training task：對每個 train query (LOO)，由 s_first 取 top-K_top=50 候選，
reranker 預測 Δ；loss = listwise_KL(s_final, s_dense) + λ_mse · MSE(Δ, qᵀe)。

Early-stop metric：Regime B 的 sem_R@10（top_k_cases=1, sem_thr=0.95），
這是論文 reranker validation 的 primary metric。

================================================================================
§7 — Phase 4 method-level extensions: scaling law + importance-weighted RVQ
================================================================================

新增的 codebook fitting workflow，runtime path 完全不變（codebook tensor 形狀
跟編碼/解碼/LUT scoring 都不動）。三步：(1) 算 absorption surface 拿 scaling
law，(2) 從 dense top-K 推導 per-case weight，(3) 用 weighted k-means 重 fit
codebook。

--- (1) Absorption surface（fit scaling law 用的 sweep）---

$PY -m diagnosis_model.cause_inference.rvq_rerank.eval_absorption_surface \
  --encoder_ckpt diagnosis_model/cause_inference/outputs/encoder_final/best_encoder.pt \
  --case_db_dir  diagnosis_model/cause_inference/outputs/case_db \
  --rvq_root     diagnosis_model/cause_inference/outputs/rvq_rerank \
  --rvq_configs  8,256 4,256 2,64 1,16 1,4 \
  --top_ks       1 2 3 5 8 10 15 20 30 50 \
  --out          diagnosis_model/cause_inference/outputs/rvq_rerank/absorption_surface.json

# 再對 raw encoder 跑一次以做 encoder-invariance 測試：
$PY -m diagnosis_model.cause_inference.rvq_rerank.eval_absorption_surface \
  --encoder_ckpt diagnosis_model/cause_inference/outputs/encoder_raw/best_encoder.pt \
  --case_db_dir  diagnosis_model/cause_inference/outputs/case_db_raw \
  --rvq_root     diagnosis_model/cause_inference/outputs/rvq_rerank_raw \
  --out          diagnosis_model/cause_inference/outputs/rvq_rerank_raw/absorption_surface.json

Power-law fit 用 numpy log-linear regression + jackknife CI（fit script 未
入庫，重做時用 /tmp/fit_absorption.py 模式即可）。fine encoder 得到
D ≈ 59.9 · ε^{0.93} / K^{0.88}，R²=0.90；fixed-effect 共用 (p, q) 後仍 R²=0.83
（ΔR² 從 fully-separate 只損失 0.006），c_raw ≈ 2 × c_fine。

--- (2) Per-case weights（drives importance-weighted codebook）---

$PY -m diagnosis_model.cause_inference.rvq_rerank.compute_rvq_weights \
  --encoder_ckpt diagnosis_model/cause_inference/outputs/encoder_final/best_encoder.pt \
  --case_db_dir  diagnosis_model/cause_inference/outputs/case_db \
  --top_k 20 \
  --out  diagnosis_model/cause_inference/outputs/rvq_rerank/case_weights.pt

產出四個 weight vector（mean=1）:
  - w_vanilla = 1（baseline）
  - w_ranking = retrieval frequency in top-20 of train queries（main contribution）
  - w_agg     = w_ranking × isolation（up-weight 孤立 case，ablation）
  - w_agg_inv = w_ranking × density（反向 ablation，negative result）

--- (3) Fit 4 variants × 5 (M,K) ---

$PY -m diagnosis_model.cause_inference.rvq_rerank.fit_rvq_variants \
  --encoder_ckpt diagnosis_model/cause_inference/outputs/encoder_final/best_encoder.pt \
  --case_db_dir  diagnosis_model/cause_inference/outputs/case_db \
  --weights_pkg  diagnosis_model/cause_inference/outputs/rvq_rerank/case_weights.pt \
  --out_root     diagnosis_model/cause_inference/outputs/rvq_rerank/variants \
  --configs 8,256 4,256 2,64 1,16 1,4 \
  --variants vanilla ranking agg agg_inv

執行完 outputs/rvq_rerank/variants/{tag}/rvq_M{M}_K{K}/codebooks.pt 即為
新 codebook，跟 vanilla 同 schema（tensor shape `[M, K, D]` + fitted flag），
可直接被 build_rvq_index.py / eval_final.py / benchmark_scale.py 載入。

論文 main claim：ranking variant 在 4/5 configs 把 K\* (damage ≤ 1pp 的最小
`top_k_cases`) 從 8/15/20/20 降到 3/10/15/15（≈33% reduction），prod regime
R@10 lift +0.2–1.2 pp。agg / agg_inv 變體 unstable，flag 為 future work
（co-retrieval graph clustering 才是 aggregation-aware 的正確結構）。
