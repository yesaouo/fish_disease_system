================================================================================
[6] Phase 4 — CRR-DeepRVQ（壓縮 + 殘差 reranker，論文 method-level 貢獻）
================================================================================
CRR-DeepRVQ = Compression-Residual Reranking for DeepSets-RVQ Case Retrieval。

把 Phase 3 已訓好的 DeepSets case encoder 凍結後，對 z_i ∈ R^768 做 post-hoc
Residual Vector Quantization 壓縮成 codes ∈ [K]^M（每 case M bytes for K≤256），
再用 query-to-candidate cross-attention reranker 學 Δ_i ≈ qᵀe_i 修正 RVQ 引入
的 ranking 誤差。

s_first(q, i) = qᵀẑ_i              # LUT scoring on RVQ codes, O(M) per candidate
s_final(q, i) = s_first + Δ(q, i)   # neural reranker on top-K_top candidates

兩個 reranker variant：
  Light  : ẑ, codes, q⊙ẑ, |q-ẑ|, s_first, ‖e‖    （compressed-memory deployment）
  Full   : 同上 + z_i + e_i                       （full-memory rerank;
                                                   analytic Δ = qᵀe 為 oracle）

論文評估在 Phase 1-aligned eval 上跑兩個 regime：
  Regime A : top_k_cases=20  → aggregation buffer 吸收壓縮損失，所有方法持平
  Regime B : top_k_cases=1   → 無 aggregation buffer，這是 method validation regime

子套件路徑：diagnosis_model/cause_inference/rvq_rerank/
產出目錄：outputs/rvq_rerank/

[6c] Stage A3 — 訓練 Light reranker（每個 RVQ config ~15s–2min）
--------------------------------------------------------------------------------
Training task：對每個 train query (LOO)，由 s_first 取 top-K_top=50 候選，
reranker 預測 Δ；loss = listwise_KL(s_final, s_dense) + λ_mse · MSE(Δ, qᵀe)。

Early-stop metric：Regime B 的 sem_R@10（top_k_cases=1, sem_thr=0.95），
這是論文 reranker validation 的 primary metric。

論文預設（M=4 K=256，輕度壓縮）：

  -m diagnosis_model.cause_inference.rvq_rerank.train_reranker \
  --encoder_ckpt diagnosis_model/cause_inference/outputs/encoder_final/best_encoder.pt \
  --case_db_dir diagnosis_model/cause_inference/outputs/case_db \
  --rvq_M 4 --rvq_K 256 \
