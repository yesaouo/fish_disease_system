  Retrieval 持平：       γ=0.75 sem R@10 = 46.4% (raw) vs 45.3% (fine-tuned)
                       γ=0.75 sem MRR  = 0.306 (raw) vs 0.302 (fine-tuned)

  Faithfulness 反轉：
    no_lesion drop:  global-type=-0.019, lesion-type=-0.031, mixed=-0.035  ← 全部負值
    no_random drop:  +0.004
    no_global drop:  +0.05

  → 結論：retrieval 可以 zero-shot（Phase 1），但 attribution 必須有 lesion-

================================================================================
[H] 端到端推論（給單張新影像）
================================================================================
完整 inference flow（給一張 query 圖）：

Step 1: RF-DETR 偵測病灶（已實作於 detection 模組）
Step 2: 對偵測到的每個 lesion crop，run VLM-Lesion fusion 取 lesion embedding
Step 3: 對全圖 run VLM-Global 取 global embedding
Step 4: 用 case encoder 取 h_final（lesions 依面積大→小排序輸入），對 case_db
        所有 h_final 做 cosine 取 top-K cases（K=20）。退路：Phase 1 hungarian
        公式（[A]），品質等價速度慢 14×。
Step 5: 對 candidate pool 中每個 cause run CEAH 得 score + α
Step 6: γ=0.75 hybrid 排序
Step 7: Diversification（cosine threshold 0.95）
Step 8: 輸出 top-N 病因 + 每個 cause 的 α breakdown（global / text / lesion）

實作可參考：
  diagnosis_model/cause_inference/eval_ceah.py 的 evaluate_one_query()
  + 加上 RF-DETR 偵測與 VLM 編碼的前處理層
