  Retrieval 持平：       γ=0.75 sem R@10 = 46.4% (raw) vs 45.3% (fine-tuned)
                       γ=0.75 sem MRR  = 0.306 (raw) vs 0.302 (fine-tuned)

  Faithfulness 反轉：
    no_lesion drop:  global-type=-0.019, lesion-type=-0.031, mixed=-0.035  ← 全部負值

[J] ABQ (Aggregation-Buffered Quantization) — case-bank codebook 壓縮（RVQ）+ 選配殘差 reranker（buffer-free 對照）
================================================================================
Phase 4 訓練見 training.txt [6]。本節提供：

雙 regime 的設計動機：Phase 1 的 cause-aggregation buffer（top_k_cases=20）吸收
所有 RVQ ranking 噪聲，所以 RVQ-only 在 production 設定下跟 dense 完全持平
（甚至 M=1 K=16 把 12780 cases 壓成 16 個 prototype 都對 R@10 沒影響）。
要 expose reranker 的價值必須拆掉 buffer：top_k_cases=1（單一 case 的 cause set
直接當預測），這也是標準 ANN benchmark 的設定。

================================================================================
[H] 端到端推論（cascade production flow）
================================================================================
Cascade = coarse-to-fine 二階段：
  Stage 1 (coarse, zero-shot):  raw SigLIP2  → case_db_raw   → top-K cases
  Stage 2 (fine, fine-tuned):   VLM-Lesion   → case_db       → cause + α

Index 雙存：
  outputs/case_db_raw/   raw SigLIP2 features        ← Stage 1 (retrieval)
  outputs/case_db/       fine-tuned VLM features     ← Stage 2 (attribution)
  兩者共享 case_id / image_id / causes / GT cause_emb_indices，差別只在 image
  feature 是否經過 VLM-Lesion / VLM-Global fine-tune。

Query-time inference flow（單張新影像）：

Step 1  [Detection]   RF-DETR 偵測病灶 bbox

Step 2  [Encode raw query features for Stage 1]
        對全圖 + 每個 lesion crop，run raw `google/siglip2-base-patch16-224`：
          raw_global   = raw_siglip2.get_image_features(whole_image)
          raw_lesion_i = raw_siglip2.get_image_features(lesion_crop_i)
        L2-normalize all.

Step 3  [Encode fine-tuned query features for Stage 2]
        對全圖 run VLM-Global（overall fine-tune）：
          fused_global = vlm_global.get_image_features(whole_image)
        對每個 lesion crop run VLM-Lesion (LocalGlobalFusionWrapper)：
          fused_lesion_i = vlm_lesion.forward_image(lesion_crop_i, whole_image)
                         = local_feat + gate * fusion_linear([local_feat, global_feat])
        L2-normalize all.

        部署優化：若記憶體允許，可只 forward base_model 一次取出 base SigLIP2
        features（這就是 raw features），再透過 LocalGlobalFusionWrapper 計算
        fused features，省一次 base forward。但 VLM-Global 是 full fine-tune（不
        是 wrapper），所以 global 端 raw vs fused 仍需兩次 forward。

Step 4  [Stage 1 — coarse retrieval]
        用 Phase 3 encoder_raw 把 (raw_global, raw_lesions sorted by area DESC)
        編成 h_query ∈ R^768，對 case_db_raw 取 top-K cases (K=20):
          top_K = top_K_argmax_j  cos(h_query, h_case_j)    [encoder_raw]
        Optional RVQ acceleration: 用 Phase 4 codebook + Light reranker (1M 規模 0.6 ms/q).
        Optional teacher 退路: Phase 1 hungarian on case_db_raw, 品質等價 14× 慢.

Step 5  [Build candidate cause pool]
        candidate_causes = union(case_db[i].causes for i in top_K)
        (~87 unique causes per query)

Step 6  [Stage 2 — fine rerank]
        對每個 candidate_cause c, 從 case_db 拿出 query 的 fine-tuned features
        (fused_global, fused_lesions) 跑 CEAH:
          score_c, alpha_c = CEAH(fused_global, text_emb_or_None,
                                  fused_lesions, cause_text_emb=c)
        其中 alpha_c ∈ R^(2+N) 是 softmax over (global, text, lesion_1..N).

Step 7  [Diversification, optional]
        如果要展示多元化 top-N，做 cosine ≥ 0.95 suppress（不影響 R@K metric）.

Step 8  [Output]
        Top-N causes ranked by score_c, each with α breakdown
        (global / text / lesion_i) for explainability.

實作可參考：
  diagnosis_model/cause_inference/eval_ceah.py 的 evaluate_one_query()
    — Step 5–8 的範本，但需把 retrieval 階段（Phase 1 hungarian）換成
      Phase 3 encoder_raw + case_db_raw.
  demo/app_gradio.py
    — 完整 cascade 的可執行 reference：Stage 1 走 raw SigLIP2 → Phase 3
      encoder_raw → Phase 4 RVQ (M=4 K=256) + Light reranker on case_db_raw，
      Stage 2 走 VLM-Lesion + CEAH on case_db，γ-free（cascade ≡ γ=0）。
      UI 另顯示各階段（detection / 編碼 / Stage1 / Stage2 CEAH）的 CUDA-synced 耗時。

注意：
  - **γ hybrid 在 cascade 下不再使用**。Cascade ≡ γ=0（純 CEAH for fine stage），
    這也是 γ-scan 中 sem R@10 + cluster R@10 最強的點（46.8% / 39.8% vs γ=0.75
    的 45.3% / 37.0%）。如果要對照 hybrid γ scan 看 contribution breakdown，
    用 [B] CEAH eval 的 --gammas 0.0 0.5 0.75 1.0 sweep.
  - Coarse stage 對 fine-tuned 跟 raw case_db 都 work（ablation 在 [A-ablation]
    / [I-ablation] 已驗證）；production 選 raw 是因為它 +1.2 pp R@10 且支持
    paper 的 zero-shot vision-language retrieval claim.
  - Fine stage 對 VLM-Lesion 是 architecturally required：raw case_db 上重訓
    CEAH 會 faithfulness 反轉（[C-ablation]）.
